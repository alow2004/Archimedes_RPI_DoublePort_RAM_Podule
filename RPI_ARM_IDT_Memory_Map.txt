Byte 0 --> 255 = Command & actions 
Byte 256 --> 1021 --> Data memory buffer
Byte 1022 --> Interrupt generated by RPI, received by ARM (/INTL) 
Byte 1023 --> Interrupt generated by ARM, received by RPI (/INTR)
  

The transmitted byte value in interrupt address act as a pointer to the command/action

Interrupting the RPI

ARM write the address of the command/action in 3FF (1023)
RPI retrieve this address, read the data and react to the generated command
During the read cycle, interrupt is cleared

RPI write the address of the command/action in 3FE (1022)
ARM retrieve this address, read the data and react to the requested command
During the read cycle, interrupt is cleared


Byte 0-255:
----------------------------------------------------------
0 - F Reserved for future use

10 - Keyboard code sent
11 - 8 bits mouse coordinates (High)
12 - 8 bits mouse coordinates (Low)
13 - Mouse buttons value
14 - I2C data            (For I2C port transmission, data are buffered starting at h100)
15 - Parallel port data  (For parallel port transmission, data are buffered starting at h100)
16 - Serial port data    (For serial port transmission, data are buffered starting at h100)
17 - Econet port data    (For econet port transmission, data are buffered starting at h100)
18 - Podule information  (String sent buffered starting at h100)
